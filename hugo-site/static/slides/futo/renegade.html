Learning peer × contract together is what catches a peer that quietly drops requests for one specific contract.

Blended conservatively with the existing router • live in Freenet since March 2026 • renegade-ml

  • Routing isn't only heuristic anymore. The network can learn from how peers have actually behaved.
  • The greedy rule is "forward to the closest peer to the contract." But closest isn't always best -- some peers are slow, overloaded, or deliberately dropping requests.
  • Renegade learns from four things per routing event: which peer, which contract, the ring distance, and time. From that it predicts whether a peer will succeed and how fast.
  • The clever part is peer-times-contract: that's what detects a peer selectively censoring one contract while looking fine on everything else.
  • It's blended conservatively with the existing router, not trusted blindly. It's been live in Freenet's routing since March. The library is renegade-ml, open source.
  • Don't go deeper into the ML unless someone asks -- the point is that this is serious engineering, not AI sprinkled on top.
