How Atlas Decentralizes

Now

One crawler, one describer, one index — we run it.

Real today

Many crawlers and many describers into the same index — different LLMs, local models, even people. The index doesn't care who produced an entry.

New work

Independent contributors — pre-authorized co-writers, or their own competing indexes anyone can run.

The goal

You choose which crawlers, describers, and rankers you trust.

Discovery with no gatekeeper — because judgment was never the network's to hold.

SAY (cues):

  • Scrupulous about real versus new work. Overclaiming would hurt most here.
  • Now: one crawler, one describer, one index, run by us. The honest starting point.
  • Real today: the index only checks signed plus well-formed, so more crawlers and describers is an off-network change.
  • A different describer is just another program that signs an entry. We literally just swapped our describer's model.
  • New work, plainly: today one key writes. Others need pre-authorizing as co-writers, or run their own competing index.
  • That open, dynamic contribution is real engineering we haven't done yet.
  • Goal: you pick which crawlers, describers, and rankers you trust. That layer is roadmap.
  • Land the line: flexible because judgment was never on the network. Integrity on the network, judgment at the edge, is how I read every app.

BACKUP (Q&A / accuracy, not for live):

  • Be explicit that pluralistic / open contribution is roadmap, not built. Today exactly one key writes the index.
