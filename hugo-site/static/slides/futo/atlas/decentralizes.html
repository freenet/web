How Atlas Decentralizes

Now

One crawler, one describer, one index — we run it.

Real today

Many crawlers and many describers into the same index — different LLMs, local models, even people. The index doesn't care who produced an entry.

New work

Independent contributors — pre-authorized co-writers, or their own competing indexes anyone can run.

The goal

You choose which crawlers, describers, and rankers you trust.

Discovery with no gatekeeper — because judgment was never the network's to hold.

  • Now walk the layers, and be scrupulous about what's real versus what's new work -- this is the slide where overclaiming would hurt most.
  • Now: one of each, run by us. That's the honest starting point.
  • Real today: because the index only checks "signed by an authorized key + well-formed," more crawlers and more describers is an off-network change. Different LLMs, a local model, even a human -- a different describer is just another program that signs an entry. We literally just swapped our own describer's model, which is the point: it's replaceable.
  • New work, say so plainly: today one key writes. Independent third parties need to be pre-authorized as co-writers, or run their own competing index -- open, dynamic contribution is real engineering we haven't done yet.
  • The goal: you pick which crawlers, describers, and rankers you trust. That pluralistic layer is roadmap -- but it's credible precisely because none of that judgment lives on the network.
  • Land the line: super-flexible, because judgment was never on the network. Then that same test -- integrity on the network, judgment at the edge -- is how I'll read every other app.
