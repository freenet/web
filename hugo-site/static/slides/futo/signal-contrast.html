If you asked me what secure messenger to use today, Signal is the obvious answer. River isn't claiming Signal's maturity — it shows the shared application state itself can live on a network nobody operates.
  • Handle this carefully and generously. Signal is excellent and mature. I use it. This is not "River beats Signal."
  • The two projects answer different questions. Signal asks: how do we run a messaging service while minimizing what the service can learn? Freenet asks: can the application live on a network nobody operates, so there's no server, no account database, no company backend at all?
