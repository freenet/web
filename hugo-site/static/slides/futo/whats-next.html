What's Next: Mail, Raven, Harvest

Mail

pre-alpha, deployed

Private asynchronous messaging. Post-quantum crypto. Extends Freenet from chat to email-shaped communication.

Raven

prototype

A social feed. Interesting because it stress-tests high-churn social state on Freenet.

Harvest

early design

A decentralized marketplace, like Etsy without the platform. Still design, not working software yet.

Not launch-ready. The point is that the same substrate is already taking on this many application shapes

SAY (cues):

  • These are next, in decreasing order of readiness.
  • Mail: pre-alpha, deployed, post-quantum. Chat to email-shaped messaging.
  • Raven: prototype social feed. Stress test, social state is high-churn.
  • Harvest: think a decentralized Etsy. Early design, not built yet, don't oversell.
  • Not all of these are ready, and that's fine. One substrate, this many shapes.

BACKUP (Q&A / accuracy, not for live):

  • Mail repo says don't store anything you'd mind losing.
  • Why not post-quantum everywhere? Keys are much bigger. X25519 public key: 32 bytes. ML-KEM-768 (lattice): 1184 bytes, ~37x. Signatures worse: Ed25519 64 B vs Dilithium ~3300 B. Bigger keys = bigger state everywhere, so PQ where it's worth it.
  • Harvest: don't spin into a trust-and-safety tangent for a marketplace that isn't built.
