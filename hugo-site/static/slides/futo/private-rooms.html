What's New: Private Rooms

Encrypted shared application state, hosted by a network that can't read it

Room secret

AES-256-GCM encrypts every message

Key distribution

ECIES (X25519) gives the secret to members only

Rotation

Weekly, and on every ban

Peers host and synchronize the room without ever seeing the plaintext. The network enforces structure and propagation, not meaning.

  • This is the headline new capability since the last talk. Private rooms show that shared, encrypted application state can live on a network nobody operates.
  • Every message is encrypted under a per-room AES-256-GCM secret. New members get that secret via ECIES -- X25519 key agreement -- so only members can decrypt.
  • The secret rotates weekly, and immediately whenever someone is banned, so a removed member can't read anything new.
  • Key idea: the peers hosting the room never see plaintext. They enforce the contract's rules and propagate updates. They don't understand the content.
  • The room is still an application object, not a row in someone's database.
