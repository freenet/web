State Governed by Code

In Freenet, applications aren't just files. They're state governed by code.

contract key = hash of the WebAssembly contract
state = the current application data
updates = signed changes the contract accepts or rejects

App-specific rules, without app-specific servers. Delegates keep your private keys and data on your own hardware.

SAY (cues):

  • Quick primer on contracts. Keep it conceptual, don't go deep unless asked.
  • A contract is WebAssembly. Its key is the hash of the code, so the governing code is baked into the address you fetch.
  • State is the app data. Updates are signed changes, and the contract decides what's valid.
  • That's app-specific rules without an app-specific server enforcing them.
  • Delegates hold your keys and private data on your own machine, so the network never sees them.
  • Delegates are what make River's private rooms and Mail possible.
