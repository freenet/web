What It Protects, and What It Doesn't

✓  It protects

  • Message content
  • Content from the peers that host the room
  • Content from removed members, after rotation

✗  It does not hide

  • That a room exists
  • Activity, timing, and message sizes
  • Membership and social-graph structure

And no Signal-style per-message ratcheting. This is shared room state with key rotation, a different design

SAY (cues):

  • Different question from Signal, not a better answer.
  • Be precise, cryptographers in the room. Protects content, not anonymity.
  • Not hidden: room exists, membership, timing, rough message sizes.
  • Not Signal's double-ratchet. Shared per-room key + rotation: a different trade-off, not a weaker Signal.
  • Double-ratchet, if useful (skip for experts): Signal re-keys every message, so a leaked key opens nothing before or after it. River: one room key that rotates, so not yet.
  • Strongest metadata + forward-secrecy for 1:1 today? Not this.
  • Signal folks here: strengths as fact. No gushing, no dig.
  • Still experimental. Proves private shared state can live on the network at all.

BACKUP (Q&A / accuracy, not for live):

  • Frame the key-rotation design as a deliberate trade-off, not a weaker version of Signal.
  • Don't claim anonymity or metadata-hiding.
