Not just AI coding — AI project management with an explicit utility function

SAY (cues):

  • Short segment, 2 to 4 minutes. Don't let it take over the talk.
  • AI not just to write code, but to decide what to work on.
  • Every issue gets a value and a cost, from Claude reading the real repo.
  • Pairwise comparisons instead of absolute scores (which drift). Priority = value minus cost.
  • Rankings go to a GitHub board. A triage skill reads it, debug-and-fix agents pick the top.
  • Framing: the board is an AI-readable control surface. Project management with a utility function, not autocomplete.

BACKUP (Q&A / accuracy, not for live):

  • Priority computed in log-odds space. Value and cost cached by the issue's content hash so it isn't re-run.
