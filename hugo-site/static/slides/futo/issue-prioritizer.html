Not just AI coding — AI project management with an explicit utility function

  • Short segment, 2-4 minutes, don't let it take over the talk.
  • We use AI not just to write code but to decide what to work on. Every open issue gets a value and a cost assessment, done by Claude reading the actual repo, cached by the issue's content hash so we don't re-run it.
  • Instead of asking an LLM for absolute scores -- which drift -- we do pairwise comparisons, which are far more stable. Priority is value minus cost, in log-odds space.
  • Those rankings get written to a GitHub project board. A triage skill reads the board, and autonomous debug-and-fix agents pick up the top-ranked issues.
  • The framing: the board becomes an AI-readable control surface. This is AI project management with an explicit utility function, not just AI autocomplete.
